# cartrend 자체 견적 엔진 전환 — Claude Code 구현 명세

## 목적
타사(아마존카) 견적 스크래핑 방식을 **폐기**하고, cartrend 자체 가격 정책으로
견적을 산출하는 독립 엔진으로 전환한다. 관리자는 차량가만 입력하면(+ 정책값은
기본 적용/필요시 조정) 모든 트림·기간·옵션 견적이 자동 계산된다.

## 전환 이유 (배경)
- 경쟁사 견적값을 실시간 수집해 자사 영업용 사이트에 노출하는 방식은
  이용약관·부정경쟁·가격 정확성 측면에서 법적 리스크가 있어 폐기한다.
- 대신 **공개된 표준 리스/렌트 계산식 + cartrend 자체 정책값**으로 견적을 낸다.
  (계산 방법론과 통상적 잔가율 범위는 업계 공통 지식이며 특정사 독점물이 아님)

## 제거 대상 (스크래핑 관련)
- `scraper/amazoncar.ts|js` (Playwright 수집기) — 삭제
- `server.js`의 `POST /api/collect` 중 아마존카 접속·수집 로직 — 삭제
- 관리자 폼의 "아마존카 URL 입력칸 / 견적 수집 버튼 / 수집 미리보기" — 삭제
- 저장된 수집값(quotes.json의 아마존카 유래 데이터) — 사용 중단(견적은 엔진이 계산)
- ※ 단, 역산/계산 수학 모듈(`scraper/fit.js`의 monthlyFromParams, calcDepositBase 등)은
  자체 엔진의 핵심으로 **재사용**한다. 이름만 중립적으로 변경 권장(예: `lib/quote-engine.js`).

## 유지·재사용 자산
- 보증금 계산식: `보증금 = floor(depositBase × 비율 / 1000) × 1000`
  - `depositBase = 차량가 − 개별소비세감면 − 제조사DC`
  - 개소세 감면은 세법 공식 자동계산 함수 유지(트림 수동입력 불필요)
- 월렌트료 계산식(balloon 잔가 모델):
  - `월렌트료(n) = (depositBase − 잔가(n)×(1+i)^-n) / 연금계수(i,n)`, `i = 연요율/12`
  - `잔가(n) = depositBase × 잔가율(n)`
- 옵션 가산: 옵션 증가분만 더하는 방식
  - `결과(n) = 기본월렌트료(n) + [월렌트료(차량가+옵션, n) − 월렌트료(차량가, n)]`

## 자체 정책값 (cartrend 기준 — 관리자 조정 가능)
아래는 **기본 정책값**이며, 관리자 설정에서 변경 가능해야 한다.
시장 통용 범위(60개월 잔가 약 40%, 요율 4~7%대)를 참고해 cartrend가 직접 결정.

### 잔가율 테이블 (기간별, 기본값 예시)
| 기간 | 잔가율(기본) |
|---|---|
| 24개월 | 63% |
| 36개월 | 56% |
| 48개월 | 49% |
| 60개월 | 41% |
| 72개월 | 34% |
| 82개월 | 28% |
- 구조: `잔가율(n) = base − slope × (n−24)/12` (base=0.63, slope=0.072/년) — 선형 근사
- 또는 위 표를 직접 테이블로 저장(기간별 명시값). **테이블 방식 권장**(직관적·조정 쉬움).
- 차급별로 다르게 두고 싶으면 차급(경/소형/중형/대형/수입/전기) 프로파일을 두고
  각 프로파일에 잔가율 테이블 매핑. (1차 구현은 단일 테이블로 시작, 차급 분기는 옵션)

### 요율 (기본값)
- 연 6.5% (기본). 관리자 조정 가능. 차급/기간별 차등 두려면 프로파일에 포함.

### 마진 정책 (선택)
- cartrend가 월렌트료에 일정 마진을 얹거나 할인할 수 있는 `marginRate`(기본 0%) 필드 제공.
- 최종 월렌트료 = 엔진계산값 × (1 + marginRate). 0이면 영향 없음.

## 데이터 구조
- 차량/트림: 기존 localStorage 카탈로그 유지(차량명, 트림명, 배기량, **차량가**, 옵션목록).
- 정책값: 별도 설정 객체(예: localStorage `cartrend_pricing_policy` 또는 data.js 상수)
  ```js
  pricingPolicy = {
    residualTable: {24:0.63, 36:0.56, 48:0.49, 60:0.41, 72:0.34, 82:0.28},
    annualRate: 0.065,
    marginRate: 0.0,
    // 차급 분기 쓸 경우:
    // profiles: { 중형:{...}, 수입:{...} }
  }
  ```
- 옵션: 트림별 옵션 목록(이름, 가격) 유지. 견적 시 선택 옵션가 합산.

## 견적 계산 흐름 (상세페이지 car.js)
1. 트림 선택 → 차량가, 옵션목록 로드.
2. 정책값(잔가율테이블, 요율, 마진) 로드.
3. 기본 월렌트료(n) = 위 공식으로 6개 기간 계산.
4. 옵션 선택 시: 옵션증가분(n)을 더함.
5. 보증금 옵션(0~50%) 각각 계산해 표시.
6. 마진율 적용(있으면).
7. 화면 출력. (스크래핑 폴백·수집값 우선 로직 제거 — 항상 엔진 계산)

## 관리자 화면 변경
- 트림 등록/수정: **차량가만 입력**(아마존카 URL칸 제거).
- 별도 "가격 정책" 설정 화면 추가:
  - 잔가율 테이블(기간별 %) 편집
  - 연 요율 편집
  - 마진율 편집
  - (선택) 차급별 프로파일 관리
- 트림 편집 화면에 **실시간 견적 미리보기**(현재 정책으로 계산된 6기간 월렌트료) 표시.

## 검증
- 차량가 67,820,000원, 기본 정책값(위 테이블, 요율 6.5%, 마진 0%)으로
  6기간 월렌트료가 합리적 범위로 계산되는지 콘솔 확인(단조감소, 양수).
- 옵션 가산 시 옵션증가분만 더해지는지 확인.
- 보증금 0~50% 단조증가, 0%는 0원 확인.
- 정책값을 바꾸면(예: 요율 6.5%→5.0%) 월렌트료가 일관되게 내려가는지 확인.

## 구현 순서 (권장)
1. `scraper/fit.js` → `lib/quote-engine.js`로 정리(중립 명칭). 순수 계산 함수만.
2. 정책값 객체·기본값 정의 + 관리자 정책 설정 화면.
3. car.js를 엔진 계산 기반으로 교체(스크래핑/수집값 의존 제거).
4. 관리자 트림 폼에서 아마존카 URL·수집 UI 제거, 실시간 미리보기 추가.
5. server.js의 /api/collect 제거(엔진은 클라이언트 계산이라 서버 불필요할 수 있음 →
   정적 사이트로 다시 단순화 가능. 단, 정책값 공유가 필요하면 최소 저장 API만 유지).

## 주의
- 카탈로그(localStorage) 차량 데이터는 절대 초기화하지 말 것. 차량가·옵션 유지.
- 기존 수집값(quotes.json)은 참고용으로 백업만 하고, 견적 산출엔 쓰지 않는다.
- 정책값 기본은 합리적 시장 범위로 두되, 최종 가격 결정권은 관리자(cartrend)에게 둔다.
